라이브러리 적용과정
    필요 > 검색 > 사용법 파악(POC proof of concept: 적용한 라이브러리의 적합성 및 사용성에 대한 실 사용 기반 평가) > 적용

    1. 필요
        특정 기능의 필요성(직접 구현하기 어려운 부분)
        구 라이브러리 → 신규 라이브러리의 적용 필요성(더 나은 라이브러리 적용)
        트랜드 및 신 기술에 대한 호기심

    2. 검색
        필요를 검색
        github/ npm trends(Node Package manager의 트랜드를 보여주는 사이트)/ 프론트엔드 커뮤니티/ 기술 블로그 참고
        유사한 라이브러리를 후보로 두고 비교(기능/ 성능/ 사이즈/ 인기/ 최신... 등의 조건 비교)
    
    3. 사용법 파악
        공식문서
        github 코드 검색(실제 사용 예시 파악)
        기술 블로그의 적용사례를 보고 분석
    
    4. 적용
        POC  적용
        프로젝트 일부에만 적용 → 검증된 뒤에 프로젝트 전체에 적용
